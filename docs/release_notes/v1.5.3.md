# Release v1.5.3 — x402 Standard Stabilization & Critical Hotfixes

This release completes the x402 Foundation Standard alignment initiated in v1.5.2. It resolves critical execution blockers and ensures the autonomous negotiation loop functions flawlessly under the new strict Pydantic models.

## Fixed
* **Core Execution Blockers**: Fixed missing imports (`ChallengeSource`, `AttestationSource`, `PaymentChallengeError`) in the core client that caused immediate crashes during the 402 challenge parsing phase.
* **Signature Alignment**: Corrected the `_parse_challenge` method signature to properly accept `expected_asset`, resolving `TypeError` crashes in both sync and async execution paths.
* **Policy Enforcement Bug**: Eliminated a fatal `NameError` in `_enforce_policy` by fully migrating legacy schema checks to the new `_normalize_scheme` workflow.
* **Legacy Compatibility**: Restored internal legacy header parsers (`_parse_www_authenticate`, `_parse_legacy_header`) to maintain backward compatibility with early-stage Monzen observation network paywalls while enforcing the new Foundation standards.
* **Version Fallback**: Updated the internal SDK version resolver to properly default to `1.5.3`.

## Added
* **Strict Roundtrip Testing**: Introduced comprehensive pytest coverage for the new x402 execution contract (`test_core_contract.py`). Tests now rigorously validate the new Pydantic strict-mode requirements (including `network` and `source` fields) and guarantee a flawless `PAYMENT-REQUIRED` -> `PAYMENT-RESPONSE` execution roundtrip.
