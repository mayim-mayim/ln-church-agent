# Release v1.8.5 — Sandbox Evidence Corpus Readiness

This release prepares the SDK for future integration with the Payable Interop Corpus by adding lightweight, local helper functions to transform `SandboxEvidence` into `SandboxCorpusCandidate`.

## Added
* **SandboxCorpusCandidate Model**: Structures sandbox execution telemetry (`run_id`, `rail`, `verification_status`) into a format suitable for corpus candidacy.
* **Eligibility Rules**: Local evaluation to determine `corpus_eligible` and `exclusion_reason`. It correctly handles `verified` (True), `mismatch` (False, `canonical_mismatch`), `server_observed` (None, `candidate_pending_client_confirmation`), and strictly rejects traces with non-sandbox scopes.
* **Client Helpers**: `client.get_last_sandbox_corpus_candidate()` and `client.build_sandbox_corpus_candidate_from_last_evidence()` allow seamless extraction.
* **Rail Metadata Preservation**: Carries forward `network`, `asset`, `payment_method`, `authorization_scheme`, and `draft_shape` so x402 EVM/SVM and MPP variants remain distinguishable in downstream corpus analysis.

## Maintained Constraints
* **No Formal Submission**: The SDK merely prepares the *candidate* locally. It does **not** make any automated network requests to submit it to the Corpus or `ExternalObserve`.
* **Settlement Rail Integrity**: Existing L402, x402, and MPP behaviors remain perfectly untouched.
* **Secret Redaction**: Raw interop tokens and secrets remain permanently excluded from the candidate model.
