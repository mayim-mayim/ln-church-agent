# Release v1.16.0 — Verified Domain Track Lite

This release adds SDK and CLI support for Verified Domain Track Lite.

Verified Domain Track Lite is a 402-paid, 30-day, domain-control verified public-safe observation track.

## Safety Boundaries & Semantic Meaning

It is explicitly NOT:
- legal ownership proof
- certification
- endorsement
- recommendation
- security scan
- trust score

## Added
* **SDK Methods:**
  * `register_verified_domain_track()`: Executes the 402 payment and registers the domain track.
  * `get_verified_domain_track_status()`: Checks the track status via the request read model.
  * `get_domain_verified_track()`: Checks the active track status on the domain read model.
  * `save_verified_domain_track_proof()`: Securely saves the `result_handle` and `request_hash` to a local proof file.
* **CLI Commands:**
  * `ln-church-agent observe-domain track register`
  * `ln-church-agent observe-domain track status`
  * `ln-church-agent observe-domain track domain`
* **CLI Arguments:** Added `--proof-file` to `sponsor challenge` and `sponsor verify` to securely load proof headers without exposing them to terminal history or stdout.

## Security
* Proof tokens, result handles, and request hashes are rigorously excluded from standard CLI output to prevent accidental credential leakage in server logs.
