# Release v1.6.2 — x402 V2 Compatibility Improvements & Payload Normalization

This release significantly improves compatibility with x402 V2 resource servers, including `accepts`-array selection, `exact` envelope construction, and extension echo-back for facilitator interoperability.

## Added
* **Transparent Protocol Extensions (`extensions`)**: The SDK now supports the x402 V2 `extensions` standard. When a Resource Server issues a 402 challenge that includes extension metadata (such as discovery configurations), the SDK captures this data and echoes it back within the final settlement payload. This facilitates upstream indexers and crawlers upon payment success.
* **Standard `exact` Scheme Support**: Integrated the standard `"exact"` scheme into the core loop, formatting `PAYMENT-SIGNATURE` headers properly for gasless V2 settlements.

## Changed
* **Autonomous Network Selection (V2 `accepts` Array)**: Upgraded `_parse_challenge` to handle multi-option `accepts` arrays. The SDK now extracts matching network options based on the agent's expected `chainId` prior to execution.

## Fixed
* **V2 Envelope Schema Compliance**: Resolved `402` loops caused by strict middleware validators. The SDK's `PAYMENT-SIGNATURE` now constructs the expected V2 envelope hierarchy: an outer `x402Version: 2`, an explicit `accepted` object, a `resource` object, `extensions` (if provided), and a nested `payload` containing the EIP-3009 cryptographic proofs.
* **Critical RPC Gas Price Fix**: Resolved a latent bug in the EVM signer where direct on-chain transfers (`lnc-evm-transfer`) would crash due to an invalid `eth_price` RPC call. The SDK now correctly calls `eth_gasPrice` and appropriately pads raw transactions with `0x`.
* **Heuristic Raw-to-Human Unit Conversion**: V2 servers typically transmit amounts as minimal-unit integers. The SDK now applies practical heuristics to normalize these values to human-readable decimals (e.g., dividing USDC values >= 100 by 1,000,000), preventing false-positive blocks by the local `PaymentPolicy`.
* **V2 Treasury & Asset Resolution**: Resolved missing `payTo` extraction from nested V2 structures and implemented a safe fallback to expected logical symbols (USDC/JPYC) when encountering raw contract addresses in the challenge.
