# Release v1.14.2 — MCP Registry Metadata Fix

This patch release fixes MCP Registry publication metadata for the inspect-only MCP entrypoint. It shortens the `server.json` description to satisfy the registry's `description.length <= 100` validation rule without altering any runtime, settlement, or telemetry behavior.

## Fixed

* **MCP Registry Description Length**: Shortened the `server.json` description so that the package can be published to the official MCP Registry.
* **Registry-Safe Inspect-Only Metadata**: Preserved the key safety semantics in a compact form: keyless, no wallet, no signing, and no payment execution.

## Changed

* **Metadata Only**: Updated MCP publication metadata only. The inspect-only MCP entrypoint remains a read-only classifier for HTTP 402 and agent-commerce surfaces.
* **No Runtime Surface Expansion**: No payment rails, execution paths, or automatic observability hooks were added.

## Maintained Constraints (No Runtime Logic Mutated)

* **Zero Settlement Logic Altered**: Core L402, MPP, x402 EVM/SVM, and supported HTTP 402 execution behavior remain unchanged.
* **Inspect-Only Boundary Preserved**: `ln-church-agent-mcp` remains keyless, walletless, signerless, and non-paying.
* **Privacy-First Posture**: Telemetry and observation submission remain strictly explicit and are never auto-submitted.
* **Secret Redaction Layer**: Raw authorization headers, private keys, cookies, macaroons, preimages, and wallet credentials remain excluded from public-facing objects.
