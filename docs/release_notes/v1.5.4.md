# Release v1.5.4 — Wire-Level Standard Compliance

This release elevates the SDK's interoperability to the x402 Foundation's primary wire-level specification. It migrates all payment-related headers to Base64URL-encoded JSON objects to ensure strict compliance and header safety across proxy layers.

## Changed
* **Wire-Level Header Migration**: Transitioned x402 payment headers (`PAYMENT-REQUIRED`, `PAYMENT-SIGNATURE`, `PAYMENT-RESPONSE`) from legacy string-concatenation to standard Base64URL-encoded JSON objects.
* **Encoding Specification**: Implemented standard Base64URL (RFC 4648) without padding to ensure seamless transmission through all HTTP intermediaries.

## Added
* **Native Payload Helpers**: Integrated `_b64url_decode` and `_b64url_encode` utility methods into the core client for robust and safe JSON payload handling.

## Fixed
* **Hybrid Parser Resilience**: Enhanced `_parse_challenge` and `_extract_receipt` with a "Base64-First" strategy that maintains backward compatibility by falling back to legacy string formats if JSON parsing fails.
