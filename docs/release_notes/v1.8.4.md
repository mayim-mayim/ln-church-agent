# Release v1.8.4 — Sponsored Access & Sandbox Evidence Alignment

This release establishes secure, local evidence tracking for Sponsored Access (Grants) and Sandbox executions, aligning the SDK with the LN Church's "Sandbox Evidence Lane" architecture.

## Added
* **SponsoredAccessEvidence Model**: Safely captures JWS grant consumption metadata (`grant_jti`, `issuer`, `scope`) with strict architectural constraints (`settlement_rail: "none"`, `access_path: "sponsored_grant"`).
* **SandboxEvidence Model**: Captures isolated testbed telemetry from `sandbox_evidence_ref.v1` and `sandbox_evidence_report.v1`, distinguishing between server-attested and client-reported receipts.
* **PaymentEvidenceRecord Extensions**: Extended the core `PaymentEvidenceRecord` with `sponsored_access` and `sandbox` optional fields. They automatically populate and safely export to your `EvidenceRepository`.
* **Public Extraction Helpers**: Added `client.get_last_sponsored_access_evidence()`, `client.get_last_sandbox_evidence()`, and `client.get_sandbox_evidence_logs(run_id)`.
* **Safe Redaction Layer**: Implemented `sha256_redacted` builder methods. Raw `interop_token` and `grant_token` secrets are permanently excluded from JSON models and evidence exports.

## Maintained Constraints
* **No External Observation Pollution**: The Sandbox Evidence Lane remains strictly internal to the testing harness (`evidence_scope: "sandbox_internal"`). The SDK **does not** automatically submit sandbox runs to the `ExternalObserve` telemetry API.
* **No Settlement Intrusions**: Existing L402, x402, and MPP execution behaviors remain completely unaltered.
* **No Pre-Mature Implementations**: Avoided premature AP2/ACP commerce protocol detections in this evidence patch.
