# Release v1.5.5 — Dual-Stack Resilience & Error Normalization

This release addresses critical parsing paradoxes introduced by Dual-Stack (L402 + x402) backend environments and hardens the client initialization sequence. 

## Fixed
* **Dual-Stack Parsing Priority**: Reordered the `_parse_challenge` logic to prioritize `WWW-Authenticate` (Lightning/L402) over `PAYMENT-REQUIRED` (x402).  This prevents the SDK from erroneously attempting EVM settlement when a valid Lightning invoice is present in a Dual-Stack response.
* **Legacy Header Supplementation**: Enhanced the legacy string-based x402 parser to automatically supplement missing `destination` data from the JSON response body, ensuring backward compatibility with older 402 challenges. 
* **Unified Error Bubbling**: Normalized `ValueError` exceptions during client initialization. Invalid private keys now return a consistent, predictable error message (`Invalid private_key format...`) regardless of the underlying cryptographic adapter (EVM or Solana).
* **Constructor Argument Propagation**: Fixed a bug in `LnChurchClient` where the `policy` and other optional parameters were not correctly passed to the `super().__init__` call, ensuring security guardrails are active by default.

## Added
* **Enhanced Base64 Resilience**: Improved internal `_b64url_decode` to more robustly handle malformed or unpadded Base64URL JSON objects from diverse proxy layers.
