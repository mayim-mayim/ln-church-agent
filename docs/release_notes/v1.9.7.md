# Release v1.9.7 — x402 Batch Settlement Inspect-Only Support

This release adds inspect-only awareness for x402 `batch-settlement`, a deferred settlement model that separates request-time voucher authorization from later batched onchain settlement.

## Added
* Detects `scheme: "batch-settlement"` in x402 `accepts[]` payment requirements.
* Classifies batch settlement as `rail="x402"` with `execution_support="observe_only"`.
* Adds deferred settlement metadata to `SettlementOption`:
  * `settlement_model="deferred_batch"`
  * `authorization_artifact="voucher"`
  * `finality_model="deferred_onchain"`
  * `requires_channel_state=True`
  * `deferred_settlement=True`
* Propagates deferred settlement metadata into MCP observation payloads.

## Safety
* `batch-settlement` is not executed.
* The SDK does not sign vouchers, deposit funds, persist channels, claim, settle, or refund.
* `_process_payment()` explicitly blocks batch settlement execution with `batch_settlement_execution_not_supported`.
* Voucher or authorization artifacts are not treated as settlement proofs.
* Observation payloads remain `payment_performed=False`, `payment_receipt_present=False`, and `verification_status="unverified"`.

## Maintained Constraints
* Existing L402 / x402 exact / SVM exact / MPP behavior remains unchanged.
* `batch-settlement` is not added to `SchemeType`.
* `batch-settlement` is not added to default `PaymentPolicy.allowed_schemes`.
