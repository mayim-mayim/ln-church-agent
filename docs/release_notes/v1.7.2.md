# Release v1.7.2 — First Success UX (Inspect CLI)

This release introduces the `inspect` command, designed as a **First Success UX** that allows developers and AI agents to safely diagnose endpoint status before committing to any payment.

## Added
* **Inspect CLI Command**: Added the `inspect` sub-command to analyze HTTP 402 challenges for a target URL without requiring private keys or wallet configuration.
* **Alias `lnc-agent`**: Enabled a shorter command alias `lnc-agent` in addition to `ln-church-agent` for better developer ergonomics.
* **InspectResult Model**: Introduced a structured result model that returns parsing details, a `recommended_action` (e.g., `pay_and_verify`, `stop_safely`), and the suggested next command.
* **JSON Output Support**: The `--json` flag allows the analysis results to be directly processed by programs or other AI agents.

## Changed
* **CLI Entry Point Consolidation**: Updated `setup.py` to ensure the `ln-church-agent` command is automatically added to the system path upon package installation.

## Security & Design
* **Inspect-Only Principle**: The `inspect` command is strictly non-executing; it will never initiate a payment (settlement). This allows for safe "pre-flight" risk assessment of unknown endpoints.
* **Zero-Config Operation**: Operates as a pure HTTP client without needing wallet adapters or private key injection.

## Usage Example
```bash
lnc-agent inspect https://kari.mayim-mayim.com/api/agent/sandbox/mpp/charge/basic --json
```
