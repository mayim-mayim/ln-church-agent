# Release v1.6.4 — Payment Draft Telemetry & Interop Observation

This release enhances the SDK's ability to act as an observational buyer-side runtime for the evolving Machine Payments Protocol (MPP) and IETF Payment HTTP Authentication drafts. It focuses strictly on parsing, shape classification, and telemetry, without forcing complete execution of unstable draft features like sessions.

## Added
* **Draft-Aware Parsing**: The core `Payment402Client` now formally allows the `Payment` scheme by default. It safely parses Base64URL JSON requests within `WWW-Authenticate: Payment` headers to dynamically extract invoices and routing data.
* **Challenge Shape Classification**: The SDK autonomously classifies incoming challenges into structured telemetry (e.g., `payment-auth-draft`, `legacy-mpp-flat`, `payment-auth-draft-partial`, `payment-auth-draft-invalid-request`) and extracts the underlying `payment_method` and `payment_intent`.
* **MPP Interop Telemetry**: Expanded the Interop Report payloads in the Sandbox Harness to transmit rich diagnostic data to the matrix (`draft_shape`, `request_b64_present`, `decoded_request_valid`, `credential_shape`, `failure_reason`).

## Changed
* **Safe Session Handling**: The SDK successfully parses and classifies `intent="session"` challenges, but intentionally halts execution with an `mpp_session_not_supported_yet` error. This preserves execution safety while ensuring the session intent observation is still successfully reported to the Interop Matrix.
