# Release v1.13.0 — Observation Provenance, Protocol Roles & Verification Cost Vector

v1.13.0 is an additive release building upon the v1.12.0 Reporter Identity Verification Layer. It introduces structured metadata for observation provenance, protocol capability roles, and verification cost vectors.

## Added
* Public-safe `observation_provenance` metadata helpers.
* `protocol_roles` shape for mapping `role` -> `protocol` -> `capability_observations`. External standard names (e.g., AP2, UCP, MCP, x402) are natively preserved.
* Optional `verification_cost_vector` helper for sdk-reported, externally checkable cost counters.
* Optional `protocol_roles` and `verification_cost_vector` fields added to explicit observation submissions (`submit_goal_attempt_observation`, `submit_external_observation`, and their async equivalents).

## Safety Boundaries & Architecture Constraints
* **Not a Trust Score**: `observation_provenance` simply aggregates the identity status at the time of reporting. `key_control_verified` is not a trust score, recommendation, verdict, or truth proof.
* **SDK-Reported Costs**: `verification_cost_vector` is explicitly `source: sdk_reported` and `not_server_metered`. It is not a billing record or a truth proof. `verification_cost_vector` is never collected or submitted automatically. It is included only when the caller explicitly supplies it to an observation submission method.
* **No `proof_id` Leakage**: Compact public read models must not expose `proof_id`, raw signatures, nonces, or private keys.
* **Read-Only Preflight**: Surface Preflight remains purely read-only and does not execute payments.
* **Preserved v1.12.0 Constraints**: `self_reported` telemetry remains fully accepted and is a first-class citizen. There are absolutely no automatic telemetry submissions and no automatic hooks to `ensure_reporter_verification()`.
