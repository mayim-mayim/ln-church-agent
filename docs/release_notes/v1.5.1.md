# Release v1.5.1 — Experimental Evidence Export/Import Layer

This update introduces an experimental, source-agnostic `EvidenceRepository` interface. It empowers the buyer-side runtime to record the basis of its payment decisions and outcomes, and reuse that evidence for future trust evaluations.

## Added
* **Evidence Repository Protocol**: Introduced the `EvidenceRepository` base class with `export_evidence` and `import_evidence` hooks (sync/async supported).
* **Payment Evidence Record**: Added `PaymentEvidenceRecord` to encapsulate the lifecycle of a 402 interaction (challenge, trust decision, receipt summary, outcome summary). Raw secrets (like preimages or private keys) are intentionally excluded from this record.
* **Execution Flow Integration**: The `Payment402Client` now automatically imports past evidence into `context.past_evidence` prior to trust evaluation, and exports the final `PaymentEvidenceRecord` upon completion or failure.
## Changed
* **Type-Safe Context Propagation**: Enhanced `ExecutionContext` with an explicit `past_evidence` field (a strictly typed `List[PaymentEvidenceRecord]`). This replaces fragile, string-based dictionary passing, allowing developers to build robust `TrustEvaluator` hooks with full IDE autocompletion and type safety while maintaining the lightweight nature of the context.
## Fixed
* **Latent Bug Fix / Error Normalization**: Squashed a sneaky latent bug in the initialization sequence. Previously, providing an invalid private key could trigger an unformatted `ValueError` directly from lower-level cryptographic adapters (like `LocalKeyAdapter`) before the `LnChurchClient` could properly handle it. These exceptions are now gracefully caught during `super().__init__` and normalized. This ensures that agents and testing environments always receive a consistent, predictable error message (`Invalid private_key format...`) regardless of the underlying wallet or environment state.

*Note: The `EvidenceRepository` is completely optional and defaults to no-op. It is designed to allow developers to seamlessly attach local databases or future reference sinks (like the LN Church API) to the agent's reasoning loop without altering the core execution logic.*
