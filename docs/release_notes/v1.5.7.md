# Release v1.5.7 — Documentation Alignment & Protocol Fluctuation Absorption

This release corrects a critical documentation misalignment introduced in v1.5.6, formally aligning the public artifacts with the SDK's core development philosophy ("The Cold Spec"). No core execution logic has been altered; the implementation was already correct.

## Changed (Documentation & Governance)
* **Protocol Fluctuation Absorption (MPP vs Payment)**: 
  Retracted the incorrect statement from the v1.5.6 release notes which claimed that the `Payment` prefix was dropped in favor of a hardcoded `MPP` prefix. 
  The SDK's underlying implementation actively and transparently supports **both** the IETF draft standard (`Payment`) and the current ecosystem reality (`MPP`). The execution engine dynamically parses the `WWW-Authenticate` challenge and constructs the `Authorization` header using the exact scheme requested by the server. 
  This update explicitly documents this behavior, fulfilling the core governance directive: *"Absorb standard fluctuations on the SDK side."*

## Fixed
* **Artifact Synchronization**: Updated `CHANGELOG.md` to strike through the misleading v1.5.6 entry, ensuring developers integrating custom downstream servers are not misled regarding IETF draft compliance.
* **Version Bump**: Updated internal SDK versioning to `1.5.7`.
