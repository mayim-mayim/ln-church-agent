# Release v1.10.2 — Capability Matrix & Standards Watchlist Alignment

This release provides strict clarity on capability boundaries and the tracking policy for emerging standards (AP2, ACP, batch-settlement, etc.). It documents the precise execution limits of the SDK without introducing any runtime behavior changes.

## Added
* **Capability Matrix (`docs/07_capability_matrix.md`)**: A human-readable mapping of all supported layers (settlement rails, commerce surfaces, observation/memory).
* **Standards Watch Alignment (`docs/08_standards_watch_alignment.md`)**: A guide on how to interpret `STANDARDS_WATCHLIST.md` and explicit rules for future implementations.
* **`get_capability_matrix()` Helper**: A safe, static, read-only helper method in `ln_church_agent.capabilities` that exposes the matrix data for downstream tooling.

## Clarified Vocabulary
Defined and standardized safe vocabulary for SDK operations:
* `classified` is not payment success.
* `observe_only` is not proof.
* `authorization_artifact` is not settlement proof.
* `verified` requires cryptographic proof, submitted tx evidence, L402 preimage, or provider receipt.

## Maintained Constraints (No Changes)
* **No payment execution behavior changed.**
* L402 / x402 / MPP / SVM exact behavior is completely unchanged.
* `batch-settlement` remains `observe_only`.
* AP2 / ACP / OKX APP remain inspect-only / guided-handoff surfaces.
* Goal Attempt telemetry remains explicit-only and is not auto-submitted.
* Observatory submissions remain opt-in only.

## Explicitly Not Added
* No AP2/ACP/APP payment execution.
* No mandate signing.
* No broker calls.
* No ACP checkout completion.
* No batch-settlement deposit/voucher/claim/refund execution.
* No `Authorization: Payment` credential generation.
* No AWS AgentCore direct integration.
* No session-intent execution.
* No recommendation engine.
