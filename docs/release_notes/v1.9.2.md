# Release v1.9.2 — Inspect-Only MCP Entrypoint

This release introduces `ln-church-agent-mcp`, a completely independent, keyless MCP (Model Context Protocol) entrypoint designed purely for inspecting HTTP 402 and Commerce Surfaces. 

## 🧭 Guided Handoff over MCP

The inspect-only MCP entrypoint now exposes v1.9.1 Guided Handoff metadata for AP2 / ACP / OKX APP-like commerce surfaces.

This allows AI orchestration frameworks to see:
* what the agent should ask the site for,
* what must not be treated as settlement proof,
* which evidence is required,
* what information is missing,
* and whether operator approval is required.

The MCP server still never signs, pays, or executes.

## 🧩 Managed Platform Boundary

This MCP entrypoint is designed to work alongside managed payment platforms, not replace them.

Managed platforms such as AWS AgentCore Payments can handle hosted wallet/session execution, payment processing, and spending guardrails. `ln-church-agent-mcp` stays on the buyer-side inspection boundary: it classifies HTTP 402 and agent-commerce surfaces, explains safe next actions, and prepares redacted observation payloads without executing payment.

No AWS-specific integration is implemented in this release.

## 🛡️ Safety Boundaries
* **Keyless Design**: This MCP server does **not** load `AGENT_PRIVATE_KEY` and does **not** require any wallet configurations.
* **Strict Non-Execution**: Even if the inspection classifies an endpoint as `pay_and_verify`, this specific MCP server will *never* attempt to sign, pay, or execute the transaction. 

## 🚀 Example Usage
You can bind this safe inspector to your LLM directly:
```bash
ln-church-agent-mcp
```
