# Release v1.6.5 — Challenge Parser Boundary Refactor

This release performs an internal refactor of the SDK's HTTP 402 challenge parsing boundary.

No public API changes were introduced.

## Changed
* Extracted challenge parsing and Payment/MPP challenge shape classification into a dedicated internal module (`ln_church_agent/challenges.py`).
* Preserved `Payment402Client` compatibility wrappers for existing parser methods.
* Kept v1.6.4 Payment draft telemetry behavior unchanged.

## Not Changed
* No changes to payment execution behavior.
* No changes to L402 / x402 / MPP settlement logic.
* No new MPP session execution support.
* No new `Authorization: Payment <base64url-json>` credential implementation.
