# Release v1.10.0 — Goal Attempt Observation & Memory

v1.10.0 introduces Day 1 Goal Attempt Observation and the Lightweight Goal Attempt Memory Read Model layer. This expands the SDK from lower-level HTTP authentication telemetry toward a public-safe, goal-conditioned attempt memory engine.

## Added

* Added `submit_goal_attempt_observation()` and `submit_goal_attempt_observation_async()`.
* Added support for `goal_attempt.v1` payloads containing goal declarations, attempt modes, sequential execution steps, optional outcome assessments, and public-safe evidence metadata.
* Added support for unassessed attempts. If `outcome` is omitted, the attempt is recorded as unassessed rather than discarded.
* Added lightweight Goal Attempt Memory read models:
  * Free Goal Attempt Summary snapshot.
  * Paid Goal Surface Candidates atomic query.
* Added `get_goal_attempt_summary()` and `get_goal_attempt_summary_async()`.
* Added `get_goal_surface_candidates()` and `get_goal_surface_candidates_async()`.
* Added compact S3 read models:
  * `kesher/read-models/goal-attempt-summary.json`
  * `kesher/read-models/goal-surface-candidates.json`

## Changed

* Bumped package and MCP server metadata to `1.10.0`.
* Preserved existing payment execution behavior. Goal Attempt submission is explicit only.
* Preserved privacy-first observation behavior. Goal Attempt telemetry is never auto-submitted.

## Pricing Tiers

* **Goal Attempt Summary:** 100% Free (No HTTP 402 challenge).
* **Goal Surface Candidates:** 1 SAT / 0.001 USDC / 1 JPYC per request.
* **Full Monzen Graph (`kesher/monzen-graph.json`):** Unchanged premium full graph tier (10 SATS / 0.01 USDC / 10 JPYC).

## Safety Boundaries

* Goal Attempt observations are not payment proofs and do not replace Monzen Traces.
* Missing `outcome` denotes an unassessed state, **never an execution failure**.
* Secret stripping recursively removes raw credentials (authorization headers, cookies, preimages, private keys, tokens) while preserving critical public metadata such as `authorization_scheme`.
* Goal Surface Candidates represent observed historical memories, **not active recommendations or verified recipes**.
* Candidate read models do not prove settlement or receipt validity.
* Runtimes and agents SHOULD utilize the summary first, candidates second, and the full graph only when deep analytical processing is required.

## Not Added (Strict Phase Bounds)

* No Recipe generation or workflow synthesis.
* No recommendation API or automated routing engines.
* No Capability Graph or `OUTPUT_CAN_FEED` edges.
* No ScoutScore integration.
* No automatic hooks into `execute_detailed()` or `execute_request()`.
