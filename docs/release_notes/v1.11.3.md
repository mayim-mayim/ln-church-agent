# Release v1.11.3 — Experimental Intent Signature Observation Sidecar

This release introduces an optional, experimental, recursively secret-stripped "Intent Signature Observation Sidecar" for explicit Goal Attempt observations.

This is designed to allow agents to attach compact, shape-only intent metadata to `goal_attempt.v1` observations, enabling Hon-den to later aggregate and reclassify them into read models.

## Added
* Optional `intent_sidecar_metadata` parameter in `submit_goal_attempt_observation()`.
* Support for `intent_signature` and `classification_claims` inside `goal` and `steps` payloads.
* Both sync and async observation paths support the sidecar recursively.

## Safety Boundaries & Constraints
* **No payment execution behavior changed.** Existing L402, x402, MPP, and SVM workflows remain untouched.
* **No automatic Goal Attempt telemetry.** Goal Attempt logging remains strictly explicit-only.
* **No stable category taxonomy.** `category_id` is treated strictly as an unverified candidate claim.
* **No recommendation, ranking, score, or routing decision.**
* **No MCP observation payload exposure.** The `build_mcp_observation_payload` explicitly ignores these experimental metadata fields.
* **Secret Stripping (Caller Responsibility Note).** Public-safe shape-only metadata is caller-supplied and recursively secret-stripped. Raw headers, tokens, preimages, API keys, and private keys will be permanently redacted before submission. However, **this release does not yet enforce a stable shape taxonomy or full entity-value redaction** (e.g., stripping raw user prompts from `target_object`). Providing strictly "shape-only" values remains the responsibility of the calling agent runtime.
