# Release v1.8.2 — Response Adapter Decompression Hotfix

Fixes `httpx.DecodingError` during `inspect` when converting `requests` responses into `httpx` responses. 
Strips stale `Content-Encoding`, `Transfer-Encoding`, and `Content-Length` headers after `requests` has already decoded the body.

Prevents external discovery workers such as OpenClaw from failing hard on malformed or mismatched response compression metadata when scouting endpoints like Alchemy x402.

No payment execution behavior changed.
