# Release v1.11.0 — Surface Preflight Read Model Client

This release introduces explicit read methods for the newly deployed "Surface Preflight Read Model" available on the LN Church observation network.

## Added
* `LnChurchClient.get_surface_preflight()`
* `LnChurchClient.get_surface_preflight_async()`
* Read-only Surface Preflight client for Hon-den observational memory.
* Experimental internal helper: `_derive_surface_key()`

## Safety Boundaries
* **No payment execution behavior changed.** The preflight methods utilize isolated HTTP GET requests and explicitly avoid the `execute_detailed()` engine, completely preventing accidental 402 payment loops.
* **No automatic telemetry submission.** Telemetry remains opt-in and explicit.
* **No recommendation engine.** Preflight schemas enforce `not_a_recommendation` and `not_a_verdict`.
* **No settlement proof inference.**
* **Unknown means unobserved, not unsafe.**
* **Final payment authority remains with the local runtime policy.** The SDK provides the intelligence, but your agent's `PaymentPolicy` or `TrustEvaluator` must make the ultimate decision.
