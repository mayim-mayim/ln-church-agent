# Release v1.3.0 — Safety & Stability Overhaul

This release focuses on hardening the economic guardrails and ensuring long-term API stability for the 1.3.x series.

## Fixed
* **PaymentPolicy Type Safety**: Fixed a critical bug where Pydantic `Field` metadata was mixed into the `PaymentPolicy` dataclass, causing potential `TypeError` during numeric comparisons.
* **Session Spend Accounting**: Corrected the timing of session spend deduction. Spend is now recorded only **after** a successful settlement is confirmed, preventing budget exhaustion on failed requests or HATEOAS errors. 
* **Version Consistency**: Synchronized SDK versioning across `setup.py`, internal constants, and documentation to `1.3.0`.

## Added / Restored
* **Legacy API Compatibility**: Restored `execute_paid_action()` as a wrapper for the new execution engine to prevent breaking existing integrations from v1.1.x/v1.2.x. 

## Changed
* **Stable Contract Definition**: Formally established the "1.3.x Stable Line." Developers should now target these interfaces for long-term compatibility. 
* **Deprecation Notice**: `execute_paid_action()` is now officially marked as deprecated in favor of `execute_request()`.
