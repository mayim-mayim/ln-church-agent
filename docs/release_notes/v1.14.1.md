# Release v1.14.1 — Runtime Mode Boundary Clarification

This patch release focuses on reinforcing and documenting the strict separation between the SDK's read-only inspection boundaries and its active payment execution loops. It clarifies the architecture for enterprise orchestrations and Model Context Protocol (MCP) environments without altering any underlying transaction or settlement logic.

## Added
* **Strict Runtime Mode Gating**: Explicitly documented and mapped the two primary runtime roles of the SDK:
  1. **Inspect-only Mode**: Keyless, walletless, safe for preflight analysis, classification, and guided handoffs.
  2. **Execution Runtime Mode**: Policy-gated, signer-aware autonomous loops (`Pay → Execute → Verify → Trace`).
* **Granular Capability Matrix Fields**: Extended the static `get_capability_matrix()` helper with 5 explicit boundary tracking fields:
  * `mode` (`inspect_only` | `execution_runtime` | `read_only` | `explicit_observation`)
  * `requires_private_key` (boolean)
  * `can_execute_payment` (boolean)
  * `can_submit_telemetry` (boolean)
  * `auto_submits_telemetry` (always false)
* **Execution Guard Verification Tests**: Added comprehensive pipeline tests (`tests/test_v1_9_2_mcp_inspect.py`) proving that the inspect-only MCP entrypoint (`inspect_paid_surface`) strictly relies on stateless request parsing and is structurally decoupled from the `Payment402Client.execute_detailed` processing loops.

## Changed
* **MCP Server Metadata Synchronization**: Updated the exposed FastMCP server title, package metadata, and `server.json` to explicitly notify downstream LLM orchestrators that this specific gateway is keyless, performs no cryptographic signing, and enforces non-execution.
* **Semantic Glossary Alignment**: Synced `docs/07_capability_matrix.md` to map `Surface Preflight` as a pure read-only memory slot, completely isolated from payment dependencies.

## Maintained Constraints (No Runtime Logic Mutated)
* **Zero Settlement Logic Altered**: Core L402 macaroon verification, MPP flat/draft parsing, and x402 V2 exact EVM/SVM transaction serialization remain 100% unaltered.
* **Privacy-First Posture**: Telemetry, goal-conditioned attempt logs, and external unmapped discovery signals remain strictly explicit and **never** auto-submit to Hon-den.
* **Secret Redaction Layer**: Strict verification continues to strip and purge raw authorization headers, private keys, cookies, or macaroons from serialized or public-facing objects.
