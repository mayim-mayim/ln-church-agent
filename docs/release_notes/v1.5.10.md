# Release v1.5.10 — The Advisor & Final Judge Architecture

This release fundamentally redefines the responsibility boundaries between the buyer-side runtime (SDK) and the LN Church backend. The SDK now consults the LN Church as an **evidence-rich advisor**, ensuring the **final decision remains strictly in the buyer-side runtime**.

## Changed (SDK)
* **RemoteTrustEvaluator**: Refactored from a centralized delegator into an evidence synthesizer. It acquires a `recommendation` and `evidence_bundle` from the remote backend and merges them with local policies (e.g., `allowed_hosts` overrides) to formulate the final `TrustDecision`.
* **RemoteOutcomeMatcher**: Transitioned into a structured verifier backend client. It collects `recommended_success` and `checks` from the LN Church, appending them to the `OutcomeSummary.external_evidence`, while allowing custom local fallback matchers to override the final verdict.
* **Fallback Mode Renaming**: Renamed the `unknown` fallback mode in `RemoteTrustEvaluator` to `allow_on_error` to explicitly match its fail-open behavior.

## Changed (Server / Monzen Network)
* **Evidence-Rich Advisory APIs**: The `/api/agent/monzen/trust/evaluate` and `/api/agent/monzen/outcome/verify` endpoints have been upgraded. Instead of merely returning a binary allow/deny or success/failure, they now return comprehensive structural checks, historical mismatch counts, and global sanctification statuses.
* **Backward Compatibility**: All new advisory APIs maintain strict backward compatibility with older v1.4.x clients by preserving legacy `is_trusted`, `decision`, and `is_success` fields.

## Architecture Notes
* **Graceful Degradation**: If the remote advisory backend is unreachable, the SDK autonomously falls back to local heuristics (`allow_on_error`, `strict` mode, or structural checks). Your agent's core execution loop will **never** stall due to a central server failure.
