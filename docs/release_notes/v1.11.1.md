# Release v1.11.1 — Auth-Capture Inspect-Only Standards Alignment

This patch aligns the SDK with emerging x402 `auth-capture` semantics while preserving the strict no-execution safety boundary.

## Added
* Inspect-only classification for x402 `auth-capture`.
* Capability Matrix entry for `x402 auth-capture`.
* Standards Watch Alignment note deferring auth-capture execution.
* Receipt vocabulary notes clarifying that receipt presence is not final settlement by itself.

## Changed
* `auth-capture` is now recognized as an x402 settlement option during inspection.
* `auth-capture` authorization signatures are explicitly treated as authorization artifacts, not settlement proofs.

## Safety Boundaries
* **No payment execution behavior changed.**
* The SDK does not sign EIP-3009 / Permit2 authorizations for auth-capture.
* The SDK does not capture, void, refund, or reclaim auth-capture payments.
* The SDK does not perform contract simulation or facilitator-specific auth-capture execution.
* Inspect-only mode remains no-payment / no-wallet / no-signer.

## Explicitly Not Added
* No auth-capture execution.
* No AP2 / ACP / OKX APP payment execution.
* No batch-settlement voucher signing or claim execution.
* No AWS AgentCore direct integration.
* No new wallet or signer behavior.
