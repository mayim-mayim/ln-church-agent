# Release v1.6.3 — MPP Charge Sandbox Harness Integration

This release expands the SDK's testing and validation capabilities by introducing a dedicated sandbox harness for the Machine Payments Protocol (MPP) `charge` flow. It enables agents and developers to rigorously prove MPP protocol compliance against the public LN Church Interop Matrix.

## Added
* **MPP Charge Sandbox Harness**: Added `run_mpp_charge_sandbox_harness()` and its asynchronous counterpart `run_mpp_charge_sandbox_harness_async()` to the `LnChurchClient`. These methods autonomously execute the `402 -> Pay -> 200` validation loop specifically against the new MPP Basic Sandbox.

## Changed
* **Extended Interop Telemetry**: Upgraded the Interop Report payloads for *both* MPP and L402 sandbox harnesses to capture protocol-specific metadata. Reports now dynamically transmit `rail` (MPP or L402), `payment_intent`, the actual `authorization_scheme` executed by the client, and `payment_receipt_present`. This provides rich observability and debugging context in the public Interop Matrix UI.
* **Dynamic Scheme Extraction**: Removed hardcoded authorization headers from the Sandbox Harness. The SDK now strictly extracts and logs the dynamically negotiated `scheme` (e.g., `Payment`, `MPP`, or custom variations) based on the server's HTTP 402 challenge.
