# Release v1.7.3 — External Observation Client & x402 Exact Diagnostics

This release introduces native support for observing and diagnosing x402 Exact Envelopes, aligning the SDK with the LN Church server's Post-Settlement Validator architecture.

## Added
* **x402 Exact Diagnostic Runners**: Added `run_x402_evm_exact_sandbox_diagnostic()` and `run_x402_svm_exact_sandbox_diagnostic()`. These runners validate the V1+V2 Hybrid challenge shape and correctly classify the rejection of unbroadcasted exact payloads as expected post-settlement validation behavior. They do *not* report fake successes.
* **External Observations API**: Added `submit_external_observation` and `get_external_observations` to separate protocol-level telemetry from standard Monzen traces. Includes strict local stripping of raw secrets (macaroons, preimages, private keys) before transmission. External Observations are protocol-level reports, not official LN Church sandbox success runs.
* **Inspect CLI Enhancement**: `lnc-agent inspect` now accurately identifies x402 exact challenges on post-settlement endpoints, returning `observe_only` to prevent unintended settlement loops.

## Changed
* **Challenge Parser Updates**: Upgraded `parse_challenge_from_response` to deeply parse the Hybrid V1+V2 challenge shape. It explicitly differentiates between logical asset symbols (e.g., "USDC") and token contract/mint addresses (`token_address`), ensuring exact payloads receive precise parameters.

## Architecture Notes
* **Post-Settlement Validation**: The SDK natively recognizes that the current LN Church x402 exact sandbox acts as a *post-settlement validator*. It accepts submitted tx hash / submitted Solana signature evidence. It does **not** broadcast unsubmitted exact payloads.
* **Expected Rejection**: The SDK diagnostic runner treats the 403 rejection of unbroadcasted payloads (e.g., "Invalid TxHash format" or "Transaction not found") as expected behavior.
* **Future Scope**: True x402 V2 exact settlement (where the facilitator broadcasts the payload) remains a future server/facilitator architectural phase.
