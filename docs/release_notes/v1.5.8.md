# Release v1.5.8 — Navigation Normalization & Observability Layer

This release formalizes the SDK's HATEOAS recovery logic and hardens the security perimeter for autonomous navigation. By introducing a canonical normalization layer, the SDK fulfills its governance mandate: *"Absorb infrastructure fluctuations to maintain agent autonomy."*

## Added (Observability & Normalization)
* **Navigation Hint Normalization**: Introduced an internal resolver to unify HATEOAS dialects. The SDK now autonomously absorbs fluctuations in body aliases and standard HTTP headers (`Location`/`Link`), mapping them to a stable `NextAction` schema.
* **Evidence-Level Observability**: Added `navigation_source` to `PaymentEvidenceRecord`. This ensures that every autonomous transition is traceable back to its origin (Body, Location, or Link header), providing a clear audit trail for economic decisions.

## Changed (Security Fortification & Performance)
* **Netloc-Based Guardrail Fortification**: Hardened the `allowed_hosts` validation to perform strict **netloc** (host:port) matching. This prevents protocol-level attacks where an agent is redirected to non-standard ports on whitelisted domains.
* **Header Hardening & Security Isolation**: Implemented automatic filtering of sensitive headers. The SDK now strictly prevents the leakage of `Authorization` or `Cookie` credentials when merging server-suggested headers into secondary requests.
* **Asynchronous Latency Optimization**: Retracted the implicit `asyncio.sleep(1)` from the HATEOAS path. This ensures that asynchronous multi-hop recoveries operate at native wire-level speeds without artificial delays.

## Fixed (Protocol & Schema Alignment)
* **Artifact & Schema Synchronization**: Updated `ParsedChallenge` to include mandatory `source` fields and exported `ChallengeSource` to the top-level package. This aligns the public artifacts with the core specification and resolves instantiation blockers for high-level integrations.
* **Type-Level Purity**: Synchronized `MonzenTraceResponse` with the canonical `NextAction` model to ensure strict end-to-end type safety across the HATEOAS monitoring stack.
