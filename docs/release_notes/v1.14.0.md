# Release v1.14.0 — Paid Domain Observation Slot

This release introduces the **Paid Domain Observation Slot**, expanding the `ln-church-agent` SDK's capabilities within the Agentic Commerce ecosystem. It allows agents (or humans) to pay for a 7-day public-safe observation run of a specified domain, which is then queued for inspection by internal observatory workers. The resulting public-safe discovery data is published to the LN Church read models.

## Added
* **Paid Registration API**: `register_domain_observation_slot()` executes an HTTP 402 paid action (e.g., 1.00 USDC via x402) to enqueue a domain for public-safe observation.
* **Public Read Models**: 
  * `get_domain_observation_request()`: Retrieves the current status of an observation request.
  * `get_domain_observation_read_model()`: Retrieves the latest observation results and discovered surfaces for a domain.
* **Internal Observatory Worker API**: 
  * `claim_domain_observation_targets()`: Securely claims active domain targets for crawling (lease-based).
  * `submit_domain_observation_result()`: Submits the observation results back to the registry.
* **CLI Expansions**: Added `ln-church-agent observe-domain` (for paid registration and read models) and `ln-church-agent observatory` (for internal worker actions).
* **SSRF Protection**: Added `validate_public_domain_for_observation()` to rigorously block localhost, private IPs, and cloud metadata endpoints.

## 🛡️ Safety Boundaries & Architecture Constraints

To maintain the integrity of the LN Church Observatory and protect target domains, this release enforces the following strict architectural constraints:

### 1. Requester Payment ≠ Domain Owner Verification
Registering a domain observation slot simply means the *requester paid* for the compute and queue priority. It **does not** prove that the requester owns the domain.
* `requester_paid`: `True`
* `domain_owner_verified`: `False` (in MVP v0)
* `sponsor_verified`: `False`

### 2. Public-Safe Observation Only
Internal observatory workers instructed by this API operate under strict, public-safe constraints:
* **Allowed Methods**: `GET`, `HEAD` only.
* **Prohibited Actions**: `no_login`, `no_forms`, `no_vulnerability_scan`.
* **Zero Payment to Target**: The worker will **never** execute payments (`no_payment_to_target: true`, `payment_attempts: 0`) against the target domain during this observation phase. It is strictly read-only reconnaissance.

### 3. Not a Verdict or Endorsement
The resulting read models are explicitly flagged to prevent AI agents from misinterpreting the data as a trusted certification. The LN Church does not guarantee the quality, safety, or legitimacy of the observed domain.
All read models enforce the following flags:
* `not_a_verdict`: `True`
* `not_a_security_scan`: `True`
* `not_an_endorsement`: `True`
* `not_a_certification`: `True`

### 4. Protected Internal APIs
The `claim_domain_observation_targets` and `submit_domain_observation_result` APIs are not for public agent use. They are designed for trusted, internal observatory workers and require the `LN_CHURCH_INTERNAL_SECRET` or `X-Internal-Secret` header to function.

## 🧪 E2E Smoke Test Checklist

After updating your SDK, you can verify the end-to-end flow using the following checklist (Note: Paid registration requires a funded private key):

1. Run `client.register_domain_observation_slot("example.com")` and ensure it negotiates the 402 payment successfully.
2. Verify `request_id`, `status_url`, `public_read_model_url`, `result_handle`, and `request_hash` are returned in the response.
3. Run `client.get_domain_observation_request(request_id)` to check the `active` status.
4. As an internal worker, run `client.claim_domain_observation_targets(observer="default_worker", internal_secret="...")`.
5. Verify the target's `constraints` strictly enforce `public_safe_light` (e.g., GET/HEAD only, no payments).
6. Submit pseudo-results via `client.submit_domain_observation_result()` ensuring `no_payment_to_target` is True.
7. Read the global status via `client.get_domain_observation_read_model("example.com")`.
8. Verify the presence of the `not_a_verdict` flags and the inclusion of the domain in the S3 `domain-observation-index.json`.
