# Release v1.5.2 — x402 Foundation Alignment & Naming Correction

This release introduces full compliance with the x402 Foundation (Linux Foundation) standards and CAIP-2 (Chain Agnostic Improvement Proposal 2) identifiers. It establishes a strict distinction between global standards and experimental custom optimizations, while prioritizing the L402 protocol as the primary execution path for autonomous agents.

## Changed
* **Standard x402 Alignment**: Formally defined the `x402` identifier as the standard protocol mode for multi-chain HTTP 402 settlements, as governed by the x402 Foundation.
* **CAIP-2 Routing Support**: Normalized the core execution logic to parse and route payments using CAIP-2 network identifiers (e.g., `eip155:137`, `solana:mainnet`) provided in HATEOAS challenges.
* **L402-First Defaults**: Reconfigured `LnChurchClient` convenience methods to utilize `AssetType.SATS` and `SchemeType.l402` as default parameters, establishing the Lightning Network as the primary settlement route.
* **Naming Correction**: Isolated custom routing implementations from standard protocols by refactoring them into canonical names:
    * `x402-direct` → `lnc-evm-transfer`
    * `x402-solana` → `lnc-solana-transfer`
    * `x402-relay` → `lnc-evm-relay`
* **MCP Cold Spec Update**: Refactored tool docstrings and argument defaults in `integrations/mcp.py` to prioritize L402 and Standard x402 for external API compliance, explicitly instructing LLMs on scheme selection matrices.

## Added
* **Alias Normalization Layer**: Introduced `_normalize_scheme()` in `client.py` to transparently map legacy non-standard identifiers (e.g., `x402-direct`) to canonical names. This ensures zero breaking changes for existing downstream integrations.
* **Compliance Verification Tests**:
    * `tests/test_alias_normalization.py`: Verifies normalization logic for legacy and standard identifiers.
    * `tests/test_client_defaults.py`: Ensures L402-first behavior and standard asset selection in the convenience wrapper.

## Deprecated
* **Legacy Enum Members**: Marked `SchemeType.x402_direct` and `SchemeType.x402_solana` as deprecated in favor of their `lnc_` equivalents.

## Fixed
* **Spec Artifacts Normalization**: Removed deprecated legacy vocabulary from `openapi.yaml`, `agent-api.json`, `SKILL.md`, and `agent-spec.md` enum definitions to ensure consistent capability discovery for autonomous parsing engines.
