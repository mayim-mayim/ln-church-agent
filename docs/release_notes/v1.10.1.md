# Release v1.10.1 — x402 Exact Token Metadata Hotfix

This release addresses compatibility issues with some x402 exact EVM payment surfaces where the `ln-church-agent` generated invalid `PAYMENT-SIGNATURE` or EIP-3009 payloads.

## Fixed
* **Header Priority**: Prefer structured `PAYMENT-REQUIRED` over `WWW-Authenticate: x402` when both are present.
* **Token Address Resolution**: Automatically resolves known token-address-only x402 exact assets to their logical asset and decimals. Implements strict case-sensitive matching for Solana Base58 mints and case-insensitive matching for EVM contract addresses.
* **Amount Normalization**: Preserves raw `accepts[].amount` for wire-level payload generation while correctly exposing human-readable amounts for policy and budget checks.
* **EIP-3009 Double Conversion**: Prevents raw/human amount double conversion in EIP-3009 exact payload generation.
* **Parameter Preservation**: Preserves structured challenge parameters such as `relayer_endpoint` from the `PAYMENT-REQUIRED` payload.

## Maintained
* **L402 / MPP Priority**: L402 / MPP / Payment draft priority in `WWW-Authenticate` remains unchanged.
* **SVM Exact**: SVM exact raw transaction behavior remains unchanged.
* **Batch Settlement**: `batch-settlement` remains observe-only.
