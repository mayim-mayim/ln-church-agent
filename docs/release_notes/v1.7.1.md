# Release v1.7.1 — Agent-Side Synthetic Corpus Replay Runner

This release adds the agent-side capabilities to read and validate the Server Synthetic Replays introduced in the Benchmark Suite.

## Added
* **Corpus Replay Runner**: Added `LnChurchClient.run_corpus_replay()` (and async counterpart) to perform a `dry-run` validation of server-side synthetic corpus replay descriptors.
* **Behavior Validation**: The runner autonomously parses the synthetic challenges and verifies if the expected behavior matches the client's internal diagnostic state (`pay_and_verify` / `observe_only` / `stop_safely` / `reject_invalid`).

## Not Added (Intentional Scope Boundaries)
* **No real payment execution**: `run_corpus_replay` operates strictly in `dry_run=True` mode. It does not hit actual payment backends or attempt to settle the synthetic payloads.
* **No raw exact wire replay**: Replays are strictly derived from normalized (`synthetic_from_corpus_v1`) descriptors provided by the benchmark server, avoiding raw wire dump vulnerabilities.
* **No MPP session execution**: Session intents correctly route to `stop_safely` observation.
* **No Authorization: Payment credential execution**.
