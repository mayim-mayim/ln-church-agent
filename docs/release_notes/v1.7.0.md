# Release v1.7.0 — Official x402 v2 SVM Exact Path

## Added
* **Official SVM Exact Path**: The SDK now recognizes `scheme: "exact"` combined with `network: "solana:<genesisHash>"` as the standard SVM exact route.
* **SVM Exact Transaction Builder**: Introduced a native transaction builder that constructs x402 v2 compatible **VersionedTransaction** payloads. It supports `SPL Token TransferChecked`, `ComputeBudget` instructions, and optional `Memo` integration with partial signature support.
* **Network-Aware RPC Resolution**: The SVM adapter now autonomously resolves default RPC endpoints for Solana Mainnet and Devnet based on CAIP-2 network identifiers.
* **Network Allowlist Policy**: Added `allowed_networks` to `PaymentPolicy` to enforce strict CAIP-2 boundary constraints.
* **Optional SVM Dependency**: Added the `[svm]` and `[all]` extra dependency flags to install required `solana` and `solders` libraries.

## Changed
* **Wire-Level Transaction Precision**: Raw `PaymentRequirements.asset` and raw minimal unit `amount` are strictly preserved and used for wire-level transaction construction. Human-readable asset normalization is used only for policy and budget evaluation.
* **Dual-Stack Key Isolation**: EVM and SVM private keys are now strictly decoupled during client initialization to eliminate parsing crashes.

## Maintained
* **Legacy Paths Unchanged**: EVM and legacy Solana (`lnc-solana-transfer`) compatibility paths remain completely unchanged and fully functional.

## Interop Validation
The v1.7.0 SVM exact path has been validated against a live Hono x402 gateway using `@x402/svm` and a Solana mainnet USDC smoke endpoint. The SDK successfully constructed a partially signed Solana `VersionedTransaction`, submitted it through the x402 v2 envelope, and received a successful `PAYMENT-RESPONSE` from the gateway.
