# Release v1.2.5 — Stable Line Maintenance & Documentation Refactoring

This release focuses on improving the developer experience and clarifying the stable API surface for the 1.2.x series. No core logic changes were made to the payment engine or settlement protocols.

## Changed
* **Standardized Example/Test Names**: Version-specific file naming has been replaced with role-based naming to ensure examples remain relevant across patch updates.
    * `examples/advanced_1_2_0.py` → `examples/advanced_receipts_and_policy.py`
    * `tests/test_v1_2_0_core.py` → `tests/test_core_contract.py`
* **README Update**: The documentation now clearly defines the "1.2.x Stable Line," helping users and AI agents understand which interfaces are guaranteed to be stable.

## Added
* **Release Note Archive**: Introduced `docs/release_notes/` to store detailed version history, facilitating easier synchronization with GitHub Releases.
