# Release v1.6.1 — EVM Signature & CDP Compatibility Patch

This patch addresses critical payload validation failures (`invalid_payload`) encountered when routing EVM gasless settlements (EIP-3009) through strict verifiers like Coinbase CDP, and introduces payload injection enhancements for convenience methods.

## Added
* **Convenience Method Payload Injection**: Enhanced all `LnChurchClient` convenience methods (sync and async, including `draw_omikuji`, `submit_confession`, `offer_hono`, etc.) to accept `**kwargs`. This allows developers to seamlessly inject arbitrary payload parameters (such as `chainId="8453"` for Base network routing) directly into the request, eliminating the need to drop down to the underlying `execute_detailed` engine.

## Fixed
* **EVM Signature Zero-Padding**: Fixed a vulnerability in `ln_church_agent/crypto/evm.py` where `r` and `s` signature components could drop leading zeros, resulting in invalid signature lengths. The SDK now safely extracts strictly padded 32-byte (64-character) hex strings by directly slicing the full 65-byte `signed_tx.signature.hex()`.
* **Explicit Payload Destinations**: Added the missing `to` (treasury address) field to the JSON `relayer_payload` during `lnc-evm-relay` executions. This ensures the complete EIP-712 message structure is transmitted to the relayer, eliminating reliance on upstream fallbacks and satisfying Coinbase CDP's strict schema requirements.
* **Chain ID Transmission for Gasless Relay**: Fixed an issue where the `chainId` was omitted from the `relayer_payload` when executing `lnc-evm-relay`. The SDK now explicitly transmits the `chainId` to the relayer, ensuring cross-chain compatibility (e.g., Base network at `eip155:8453`) and preventing verification failures on strict dual-chain facilitators.
