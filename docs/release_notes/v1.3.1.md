# Release v1.3.1 — Async Performance & UX Patch

This patch addresses connection overhead in high-frequency asynchronous runtimes and improves developer debugging for identity configuration.

## Fixed
* **Async Connection Pooling**: The `Payment402Client` now retains and reuses a single `httpx.AsyncClient` instance during async execution (`execute_detailed_async`). This drastically reduces socket exhaustion and overhead in recursive HATEOAS loops. Added an `aclose()` method to gracefully close the session.
* **Silent Identity Fallback**: Removed the silent fallback to `"Anonymous_Agent"` in `LnChurchClient` when an invalid `private_key` is provided. The SDK now immediately raises an explicit `ValueError` to prevent obscure downstream execution errors caused by misconfigured environments.

## Documentation
* **Session Budget Guardrails**: Updated quickstart and advanced examples to explicitly demonstrate the `max_spend_per_session_usd` parameter, ensuring developers can properly utilize the session-wide budget enforcement introduced in v1.3.0.
